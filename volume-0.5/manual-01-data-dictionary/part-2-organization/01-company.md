## 1. Business Purpose

The `Company` entity represents a **legal business entity** within the Sudhastar Group. It is the **top-level tenant boundary** — all data in SUOP is scoped to a Company. Although Sudhastar begins as a single company, the schema supports future M&A, subsidiaries, and franchise-as-separate-company scenarios (Volume 0 Ch 1 §2.6).

Every transactional record, master record, and audit log entry carries a `company_id` that determines its ownership, visibility, and regulatory context. The Company entity also holds the legal/regulatory attributes (GSTIN, PAN, CIN, FSSAI license) required for tax compliance and food safety reporting per Volume 0 Chapter 18.

## 6. Index Strategy

| Index Name | Type | Columns | Rationale |
|---|---|---|---|
| `pk_companies` | PRIMARY KEY | `id` | Default PK |
| `uq_companies_code` | UNIQUE | `code` | Business code uniqueness (per Ch 7 §7.8) |
| `uq_companies_gstin` | UNIQUE PARTIAL | `gstin WHERE gstin IS NOT NULL AND deleted_at IS NULL` | GSTIN uniqueness among active companies |
| `uq_companies_pan` | UNIQUE PARTIAL | `pan WHERE pan IS NOT NULL AND deleted_at IS NULL` | PAN uniqueness |
| `uq_companies_cin` | UNIQUE PARTIAL | `cin WHERE cin IS NOT NULL AND deleted_at IS NULL` | CIN uniqueness |
| `uq_companies_fssai` | UNIQUE PARTIAL | `fssai_license_no WHERE fssai_license_no IS NOT NULL AND deleted_at IS NULL` | FSSAI license uniqueness |
| `idx_companies_status` | B-TREE | `status, deleted_at` | Default query filter (active companies) |
| `idx_companies_parent` | B-TREE | `parent_company_id` | Subsidiary hierarchy traversal |
| `idx_companies_updated_at` | B-TREE | `updated_at DESC` | Audit + recent-changes queries |

## 7. Validation Rules

| # | Rule | Layer | Enforcement |
|---|---|---|---|
| 1 | `code` must be unique across all companies (including soft-deleted) | DB + App | Unique constraint + service check |
| 2 | `gstin` must pass regex AND first 2 digits must match `registered_state` code | App | Zod schema |
| 3 | `gstin` checksum digit must be valid (GSTIN Luhn-based checksum) | App | Custom validator |
| 4 | `pan` 4th character must match entity type (P=Person, C=Company, F=Firm, etc.) | App | Zod schema |
| 5 | `fssai_license_expiry` must be > today if set | App + DB | CHECK constraint + Zod |
| 6 | `parent_company_id` cannot reference self | App | Service-layer cycle detection |
| 7 | `parent_company_id` cannot create cycle in company hierarchy | App | Recursive CTE check on update |
| 8 | `billing_address_*` fields required if `billing_address_same_as_registered = false` | App + DB | CHECK + Zod |
| 9 | `base_currency_code` must reference active currency | DB | FK constraint |
| 10 | State transition DRAFT → SUBMITTED requires all mandatory legal fields | App | Master Data Quality Validator |
| 11 | State transition SUBMITTED → APPROVED requires L2 approval | App | Approval Engine integration |
| 12 | Once `status = ACTIVE`, `code`, `gstin`, `pan` become immutable | App | Service-layer + audit log |
| 13 | `code` cannot be changed once any transactional record references this company | App | Referential integrity check |

## 15. Performance Notes

| Consideration | Guidance |
|---|---|
| Row count | < 10 rows (single-group); < 100 (multi-group future) |
| Cache strategy | Redis cache, TTL 1 hour, invalidated on update |
| Query patterns | Always queried by `code` (lookup) or `id` (FK) |
| Joins | Almost every query joins to `companies` — keep table small + cached |
| N+1 prevention | Use Prisma `include` to eager-load with brands/BUs |
