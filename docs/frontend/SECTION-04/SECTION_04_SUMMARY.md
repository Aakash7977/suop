## 2. Business Objective

Section 04 serves as the **operational execution layer** of the ERP. While Section 03 (Master Data) defines WHAT entities exist (products, customers, suppliers, warehouses), Section 04 defines HOW those entities move, transform, and are tracked through the supply chain.

**Core objectives**:
1. **Inventory Accuracy** — Real-time stock visibility across all warehouses, bins, batches, and lots
2. **Operational Efficiency** — Directed putaway, optimized picking, FEFO enforcement, wave planning
3. **Compliance & Traceability** — Batch/lot genealogy, FEFO/FIFO, expiry management, recall capability
4. **Cost Control** — Moving average cost, landed cost allocation, GL postings, revaluation workflow
5. **Resource Optimization** — Workforce allocation, equipment utilization, SLA monitoring
6. **Yard & Dock Management** — Truck queue, dock scheduling, gate control, cross-docking

## 3. Business Process

### Inbound Flow (Procurement → Receiving → Putaway)
```
Purchase Order → ASN (Advanced Shipping Notice) → Gate Entry → Dock Assignment
→ Unloading → Quality Inspection → Goods Receipt (GRN) → Putaway Task → Bin Confirmation
→ Inventory Updated (Stock-In) → Moving Average Cost Recalculated → GL Posted
```

### Outbound Flow (Sales Order → Fulfillment → Dispatch)
```
Sales Order → Inventory Allocation → Wave Planning → Picking Task → Barcode Picking
→ Packing Station → Quality Check → Shipping Label → Dispatch Planning → Vehicle Assignment
→ Loading → Seal Vehicle → Gate Exit → Delivery → Proof of Delivery (POD)
```

### Internal Flow (Stock Movements)
```
Stock Issue (production/sales/scrap/damage) → Stock Transfer (warehouse/bin) →
Stock Adjustment (correction) → Cycle Count → Variance Analysis → Approval → Ledger Posted
```

### Batch & Expiry Flow
```
Batch Created → FEFO Priority Calculated → Near-Expiry Alert → Expiry Action
(Discard/Discount/Donate/Return) → Recall (if needed) → Genealogy Trace (forward/backward)
```

## 4. Business Rules

### Inventory Rules (Sprint 12)
- Available = Received − Issued − Reserved − Damaged
- 18 transaction types (PURCHASE_RECEIPT, PRODUCTION_ISSUE, SALES_ISSUE, TRANSFER, ADJUSTMENT, etc.)
- Ledger entries are IMMUTABLE (is_immutable = true)
- Reversals create new entries with negative deltas
- Opening Stock, Scrap, Expiry are NOT reversible
- Moving Average Cost = (prevQty × prevMAC + newQty × unitCost) / totalQty

### Goods Receipt Rules (Sprint 13)
- 10 receipt types (PURCHASE_RECEIPT, MANUFACTURING_RECEIPT, SALES_RETURN, etc.)
- Quality hold separates receiving from putaway
- Stock NOT available until quality release + putaway completion
- 10% over-receipt tolerance (backend: goods-receipt/service line 30)
- Auto PO balance update on GRN posting

### Stock Issue Rules (Sprint 14)
- 11 issue types (PRODUCTION_ISSUE, KITCHEN_ISSUE, SALES_ISSUE, SCRAP_ISSUE, etc.)
- 6 picking strategies (FIFO, FEFO, NEAREST_BIN, WAVE, ZONE, PRIORITY)
- Picking required before issue
- Auto-posts to inventory ledger

### FEFO/FIFO Rules (Sprint 19)
- FEFO Priority 1–5: expired/critical (<7 days)
- Priority 6–20: near expiry (7-15 days)
- Priority 21–50: approaching (15-30 days)
- Priority 51+: healthy (>30 days)
- Action triggers: FEFO_PRIORITIZE, DISCOUNT, DONATE, DESTROY, RETURN_TO_SUPPLIER

### Costing Rules (Sprint 20)
- 6 costing methods: FIFO, WEIGHTED_AVERAGE, MOVING_AVERAGE, STANDARD, ACTUAL, SPECIFIC_IDENTIFICATION
- 8 landed cost components: FREIGHT, INSURANCE, CUSTOM_DUTY, LOADING, UNLOADING, TRANSPORT, HANDLING, BROKERAGE
- 5 allocation methods: QUANTITY, WEIGHT, VOLUME, VALUE, EQUAL
- Revaluation workflow: DRAFT → PENDING_APPROVAL → APPROVED → POSTED
- ABC class: A = top 20% SKUs / 80% value, B = 30% / 15%, C = 50% / 5%

### Putaway Rules (Sprint 25)
- 5-factor bin scoring: Capacity (20%), Distance (30%), Product Compatibility (15%), Temperature Match (20%), Picking Efficiency (15%)
- System decides bin, not operator (Chief Architect recommendation)

### Dispatch Rules (Sprint 27)
- 8-step Vehicle Load Verification chain
- 9 dispatch types, 5 vehicle types, 4 ownership models
- 4 seal types: BOLT, CABLE, ELECTRONIC, TAMPER_PROOF

### Cycle Count Rules (Sprint 18)
- ABC strategy: Class A = daily, B = weekly, C = yearly
- Blind count hides system qty from counter
- Approval chain: Supervisor → Warehouse Manager → Finance
- 6 variance types: MISSING, EXTRA, WRONG_LOCATION, WRONG_BATCH, WRONG_UOM, WRONG_PRODUCT

6. **Implement phase by phase** — Follow the Sprint order (12→32). Complete each Sprint before moving to the next.

### Business Recommendations

1. **Prioritize inventory accuracy** — Sprint 12 (Inventory) is the foundation. Everything depends on it.

2. **Enforce FEFO for food manufacturing** — Batch & Expiry (Sprint 19) is critical for food safety compliance (FSSAI, HACCP).

3. **Directed putaway over operator choice** — System should decide bin placement using 5-factor scoring, not the operator.

4. **Two-stage barcode verification** — Pick: Scan Bin→Product→Batch→Tote. Pack: Second-scan verification. This prevents 99% of picking errors.

5. **Vehicle Load Verification chain** — 8-step verification before gate exit prevents dispatch errors.

6. **Cycle count by ABC class** — Count Class A daily, B weekly, C yearly. Don't count everything at once.
