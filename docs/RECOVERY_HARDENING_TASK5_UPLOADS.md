# Task 5: Binary Uploads Removed from Git

**Status**: ✅ COMPLETE — 1,320 runtime files removed from git tracking

## Problem

The git repository was tracking 3 categories of runtime/binary files that should never be in version control:

| Category | Files in Git | Size | Why it was wrong |
|---|---|---|---|
| `apps/backend/uploads/` | 29 | 61 MB | Runtime user uploads (COA PDFs, evidence photos, supplier documents). Belongs in S3 / persistent volume. |
| `apps/backend/coverage/` | 50 | ~5 MB | Generated test coverage HTML reports. Regenerated on every test run. |
| `apps/backend/db/` | 1,241 | 41 MB | PostgreSQL data directory (binary table files, WAL logs, etc.). Runtime state. |

**Total: 1,320 files (~107 MB) of runtime artifacts polluting the git history.**

## Action Taken

### 1. Removed from git index (files preserved on disk)
```bash
git rm --cached -r apps/backend/uploads/    # 29 files
git rm --cached -r apps/backend/coverage/   # 50 files
git rm --cached -r apps/backend/db/         # 1,241 files
```
Files remain on disk — only the git tracking was removed. No data loss.

### 2. Updated `.gitignore`
```
# Runtime uploads — never commit user-uploaded files
apps/backend/uploads/
apps/backend/uploads/**
!apps/backend/uploads/.gitkeep    # preserve dir structure

# PostgreSQL data directory — runtime state, never commit

# Build output
apps/backend/dist/
dist/

# Coverage reports (expanded existing rule)
/coverage
apps/backend/coverage/
apps/backend/coverage/**
```

### 3. Added `.gitkeep` to preserve uploads directory
Created `apps/backend/uploads/.gitkeep` (force-added with `git add -f`) so the directory structure is preserved for new clones. The directory is empty in git but ready to receive uploads at runtime.

## Recommendation for Production

In production, uploads should be stored in S3 (or compatible object storage), not on the local filesystem. The backend's `FileService` (in `core/files/file-service.ts`) already supports S3 via the `S3_ENDPOINT`, `S3_BUCKET`, `S3_ACCESS_KEY`, `S3_SECRET_KEY` env vars. The local `uploads/` directory is a development fallback only.

For development with persistent uploads across container restarts, mount a volume:
```yaml
# docker-compose.yml
services:
  backend:
    volumes:
      - ./apps/backend/uploads:/app/apps/backend/uploads
```

## Git History Note

The 1,320 files are removed from the **current commit forward**. They still exist in git history (in previous commits). To permanently purge them from history (e.g. to reduce repo size), use `git filter-repo` or `BFG Repo-Cleaner`. This is **not** done automatically because it rewrites history and requires force-push coordination.
